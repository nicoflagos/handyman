# Data Breach Notification Process (Draft)

**Effective Date:** [DATE]

## 1) Trigger
Any confirmed or reasonably suspected exposure of:
- ID images or ID numbers
- wallet/transaction records
- authentication tokens
- push device tokens linked to identities

## 2) Steps
1) Confirm breach scope and affected users
2) Contain and remediate (rotate credentials, revoke tokens, patch)
3) Prepare notification content (what happened, what data, what to do)
4) Notify:
   - impacted users by in-app notice/email (if configured)
   - payments partner (if payments impacted)
   - regulators/law enforcement if required by law (via counsel)

## 3) Notification content
- incident date range
- type of data affected
- protective steps users should take
- support contact and timeline for follow-ups
