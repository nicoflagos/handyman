# Chargeback & Refund Policy (Draft)

**Effective Date:** [DATE]

## 1) Scope
This policy covers disputes initiated via the payments partner (chargebacks) and platform-led refunds.

## 2) Preventive controls (current product)
- escrow funding only at job start (with start code + before-image)
- completion requires completion code + after-image
- in-order chat provides a record of fee confirmation and scope agreement

## 3) Handling process
When notified of a chargeback/refund event:
1) freeze relevant payout until resolved (if funds still held)
2) collect evidence: order timeline, chat, images, verification code events
3) respond to partner within [X] days

## 4) Outcomes
- upheld chargeback: partner debits as required; platform updates ledger
- rejected chargeback: funds remain settled; platform closes case

## 5) Customer service refunds
Platform can issue goodwill refunds subject to management approval and partner capabilities.
